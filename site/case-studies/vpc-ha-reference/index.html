Case Study

VPC HA Reference — ALB + Multi-AZ + RDS

Multi-AZ VPC with public/private subnets, NAT, ALB, EC2 ASG, RDS MySQL, CloudWatch alarms, and SNS ops alerts.

Context

Target: a reusable baseline for reliable web apps with strict separation between public and private tiers and automated failover characteristics.

Architecture

  • Two or three AZs; public subnets for ALB + NAT, private subnets for ASG + RDS
  • Security groups and NACLs to enforce least privilege
  • CloudWatch alarms → SNS; SSM for parameters; Systems Manager for patching

Decisions & Trade-offs

  • RDS Multi-AZ: higher cost, but 99.99% DB SLA.
  • NAT gateways per AZ: improves resilience; increases cost.
  • ASG warm pools: faster scale-out vs additional instance cost.

Results

Uptime 99.9% → 99.99% Cost/mo $1,240 → $410 MTTR ↓ 40%

Cost & Ops

  • Use Savings Plans or reserved instances for steady ASG capacity.
  • Right-size instance families; consider Graviton.
  • Enable RDS storage autoscaling + Performance Insights.

Improvements

  • Introduce AWS Backup vault + cross-region copy.
  • Blueprint CDK stack with parameterized CIDRs and AZ count.
  • Blue/Green for RDS upgrades.
