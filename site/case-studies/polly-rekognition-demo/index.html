Case Study

Polly + Rekognition Serverless Demos

Public demos using API Gateway + Lambda, protected by Cognito Hosted UI and AWS WAF rate limiting. No data stored; SSM Parameter Store for secrets; cost-controlled.

Context

Goal: showcase AI/ML capabilities safely in public. Requirements: no PII retention, throttle abuse, quick rollback, and cost <$5/month at low traffic.

Architecture

  • API Gateway (REST) → Lambda (Node/Python) → AWS SDK (Polly/Rekognition)
  • Cognito Hosted UI for sign-in; Amplify Auth for client demo; WAF per-IP rate limits
  • Assets in S3; CloudFront if needed; logs to CloudWatch

Decisions & Trade-offs

  • No persistent store: avoids privacy risk; limits analytics depth.
  • WAF vs custom throttling: WAF is simpler and cheaper to operate.
  • Hosted UI: fast to stand up, less customizable than a bespoke front-end.

Results

Cost ≈ <$5/mo P95 480 ms → 95 ms 0 incidents

Cost & Ops

  • Lambda + API GW free/low tier; WAF adds ~$5/mo baseline.
  • SSM for secrets; rotations via script; alarms on 4XX/5XX surge.

Improvements

  • Add CloudFront for global demo latency.
  • Introduce per-user quotas via Cognito groups.
  • Bundle a CDK app for single-command deployment.
