<div class="eyebrow">Case Study</div>

<h1 class="h1">VPC HA Reference — ALB + Multi‑AZ + RDS</h1>

<p class="sub">Multi‑AZ VPC with public/private subnets, NAT, ALB, EC2 ASG, RDS MySQL, CloudWatch alarms, and SNS ops alerts.</p>

</section>

<section class="container layout">

<article class="content">

<div class="card">

<h2>Context</h2>

<p>Target: a reusable baseline for reliable web apps with strict separation between public and private tiers and automated failover characteristics.</p>

</div>

<div class="card">

<h2>Architecture</h2>

<ul>

<li>Two or three AZs; public subnets for ALB + NAT, private subnets for ASG + RDS</li>

<li>Security groups and NACLs to enforce least privilege</li>

<li>CloudWatch alarms → SNS; SSM for parameters; Systems Manager for patching</li>

</ul>

<figure class="card" style="padding:0"><img src="/assets/diagrams/vpc-ha-reference.png" alt="VPC architecture with ALB, ASG, NAT, RDS across AZs" style="max-width:100%;border-radius:14px"/><figcaption class="kbd"style="padding:12px">Diagram placeholder.</figcaption></figure>

</div>

<div class="card">

<h2>Decisions & Trade‑offs</h2>

<ul>

<li><strong>RDS Multi‑AZ:</strong> higher cost, but 99.99% DB SLA.</li>

<li><strong>NAT gateways per AZ:</strong> improves resilience; increases cost.</li>

<li><strong>ASG warm pools:</strong> faster scale‑out vs additional instance cost.</li>

</ul>

</div>

<div class="card">

<h2>Results</h2>

<div class="metrics"><span class="metric">Uptime 99.9% → 99.99%</span><span class="metric">Cost/mo $1,240 → $410</span><span class="metric">MTTR ↓ 40%</span></div>

</div>

<div class="card">

<h2>Cost & Ops</h2>

<ul>

<li>Use Savings Plans or reserved instances for steady ASG capacity.</li>

<li>Right‑size instance families; consider Graviton.</li>

<li>Enable RDS storage autoscaling + Performance Insights.</li>

</ul>

</div>

<div class="card">

<h2>Improvements</h2>

<ul>

<li>Introduce AWS Backup vault + cross‑region copy.</li>

<li>Blueprint CDK stack with parameterized CIDRs and AZ count.</li>

<li>Blue/Green for RDS upgrades.</li>

</ul>

</div>

</article>
